# 前端架构设计方案 — 输出模板

> 本文件定义架构方案的标准输出结构。
> Agent 应按此模板输出，根据复杂度 Level 裁剪输出范围。
>
> **产出路径规则**：如果目标项目已有目录，写入 `{项目根}/docs/frontend-architecture.md`；如果无真实项目目录（试跑/诊断），写入 skill 包的 `outputs/{项目名}-frontend-architecture.md`。除非用户指定其他路径。

## 12. 后续编码建议

### 编码前

1. 按本方案的目录结构创建项目骨架
2. 配置组件库、样式方案、状态管理
3. 实现 AppShell 和路由框架
4. 建立请求层和错误处理

### 编码中

1. 每个功能开发前检查是否在 MVP 范围内
2. 新增组件遵循分层原则（UI 组件 / 业务组件 / 页面组件）
3. 样式使用 Token，不硬编码颜色和间距
4. 命名遵循 AI 友好规则

### 编码后

1. 运行 `decision-checklist` 自检
2. 架构变更记录在下方"架构变更记录"区
3. 重大结构变更时重新评审复杂度

「简」= 给方向不展开（如组件库只写选择和一句话理由，不写排除方案对比）。
「—」= 该 Level 不输出该章节，但保留标题并标注"本项目复杂度较低，此项暂不展开"。

### 子工作流局部输出指引

当用户意图是局部问题（如"帮我选组件库""目录结构怎么规划""MVP 第一版做哪些"）时，不输出完整 12 项结构。局部输出格式：

1. **一句话复杂度判定**：Level [X]（决定性维度：[维度]），用于确定输出深度
2. **仅输出相关章节**：按 `core/workflow.md` §8 路由表的"输出项"列，只输出对应的编号章节
3. **不输出其他章节**：不要保留标题并标注"暂不展开"，直接省略
4. **不使用完整 YAML frontmatter**：局部输出不需要完整的 11 字段 frontmatter
